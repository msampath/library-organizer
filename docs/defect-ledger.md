# The Defect Ledger

This is the contract of this codebase. Every entry is a **real failure** â€” from the
five-pipeline consolidation this project distills (v1â€“v5, Jun 2026), its closeout
(2026-07-06), or this engine's own adversarial multi-agent review. Every mechanism exists
to make one of these *structurally impossible* â€” unrepresentable or machine-refused â€” and
each entry names the regression test that keeps it dead.

Rules:
- **Append-only.** New incidents get new entries.
- **Do not weaken a mechanism without consulting its entry here.** If a change makes a
  ledger scenario possible again, the change is wrong, whatever else it improves.
- Every `tests/â€¦::test_â€¦` named below is enforced to exist by
  `tests/test_defect_ledger.py` â€” a ledger citation that goes stale fails CI. That test is
  the repo's brand made executable.

## Part 4 â€” v0.2 router hardening

v0.2's hierarchical router must satisfy a hard user constraint: repairing a library may
**never move already-properly-organized trees**. The pre-review traces found four ways
the first implementation broke that (or its supporting identity machinery); each is
fixed and pinned. `naming.py` itself is the standing answer to L3 â€” the year parser
accepts only parenthesized, plausible years, permanently
(`test_naming.py::test_year_only_in_parentheses`).

| # | What the trace found | The fix | Named regression test |
|---|---|---|---|
| C11 | A language tag in the FILENAME outranked the folder the file lived in: `Movies/Tamil/Padayappa (1999)/Padayappa.(1999).English.Subs.mkv` routed to `Movies/English/â€¦` â€” a correctly-placed file would move. | Language detection is **positional**: a directory segment that IS a configured language beats directory tokens, which beat filename tokens; hints and the explicit default come after. | `test_router.py::test_filename_language_tag_never_outranks_the_folder` |
| C12 | A track directly under `Music/<Language>/` re-nested forever: the album-context tail treated the language folder as an album â†’ `Music/Tamil/Tamil/song.mp3`. | An already-placed rule for anything under `Music/<Language>/`, plus language-segment suppression in the tail for new placements. | `test_router.py::test_music_directly_under_language_folder_does_not_nest` |
| C13 | Valid hand-named homes were second-guessed: `Friends (1994)/Season 05/` re-derived to `Friends/Season 05/`, and a movie folder whose name differed from the filename parse re-derived into a new folder. | Already-placed rules run FIRST: a Season-folder home under the TV root, or a movie folder whose `(Year)` matches the file, is kept verbatim. The router improves the unorganized; it never re-derives valid structure. | `test_router.py::test_valid_hand_named_homes_are_never_second_guessed` |
| C14 | `plan_id` hashed the header's `created` timestamp, so "identical rebuild â†’ same plan" (which C3's executed-preservation depends on) was true only within the same clock second â€” a timing-dependent identity. | `plan_id` is semantic: a hash over (kind, source, config, inputs, rows) excluding the timestamp; `content_sha256` remains as separate byte-level file integrity. | `test_regressions.py::test_rebuild_of_executed_plan_keeps_executed` |

The repair contract as a whole â€” scoping is a hard boundary, correct trees produce zero
rows, the loop converges â€” is pinned end-to-end by
`test_reorganize.py::test_reorganize_moves_flat_mess_and_spares_correct_trees` and
`test_reorganize.py::test_scoping_is_a_hard_boundary`.

## The meta-entry

**L0 â€” every version of the predecessor existed to clean up the one before it.** The root
defect was that safety lived in human discipline. The mechanism that kills L0 is this
codebase's architecture: one kernel, one store, one config, machine-refused staleness, and
this ledger â€” enforced by `test_architecture.py` (no module but the kernel mutates the
filesystem) and `test_defect_ledger.py` (every entry above has a real, named test).

## Part 5 â€” v0.2 correctness review (the conservatism posture)

An adversarial review of the v0.2 router reproduced three more violations of the hard
constraint, sharing one root cause: the already-placed guards were too narrow and
case-brittle. The fix is a posture, now load-bearing: **a file already under its media
root stays there unless an explicit, evidence-backed repair applies** â€” and every
root/segment comparison is case- and separator-insensitive.

| # | What the review reproduced | The fix | Named regression test |
|---|---|---|---|
| C15 | The guards only recognized specific shapes: an `Artist/Album` music tree flattened to `Other/`, photo albums (`Wedding 2019/`) scattered to `Unsorted/`, and a config root typed in different case (`video/movies`) re-armed every fixed idempotence bug on Windows. | Anything under `music_root` stays, period. Under `photos_root` only two evidence-backed repairs move a file (`Unsorted/`â†’year with EXIF; a year folder contradicted by EXIF). Series-level groupings under `tv_root` (incl. `Specials/`) are trusted. All comparisons casefold + separator-normalize (`_ci`/`_inside`). | `test_router.py::test_artist_album_trees_are_never_flattened`, `test_router.py::test_photo_albums_are_never_scattered`, `test_router.py::test_layout_roots_compare_case_insensitively`, `test_router.py::test_specials_and_series_level_groupings_are_trusted` |
| C16 | `build_organize` hard-refused the whole plan (exit 2) on the first same-basename routed collision â€” `IMG_0001.jpg` in two camera folders made a photo source unplannable, one pair per attempt. | Colliding content-routed rows demote to the provenance-flat destination (unique by construction) and are counted in the notes; the plan builds. `_rows_unique_dsts` remains the final backstop. | `test_plan.py::test_organize_same_basename_collision_falls_back_flat` |
| C17 | `build_reorganize`'s occupied-destination check was byte-exact while the filesystem is case-insensitive: a case-variant twin planned a move that drifted forever (`plan â†’ skipped_drift â†’ same plan`), breaking convergence. | Occupancy is tested against the normcased index (collected in the same pass); case-variant twins stay put and are reported. | `test_reorganize.py::test_case_variant_occupied_destination_converges` |
